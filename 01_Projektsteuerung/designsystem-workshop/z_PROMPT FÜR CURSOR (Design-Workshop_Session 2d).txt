Web-Recherche zu "Information Architecture Layers" zeigt:
- "Separation of Concerns" ist Standard in IA
- Content → Semantic → Presentation ist etabliertes Modell
- Gilt für alle Content-Websites (Kurse, Docs, News)

Quellen erwähnen:
- "Content Modeling" (strukturierter Content-Ansatz)
- "Semantic Layer" (Bedeutung/Funktion von Daten)
- "Presentation Layer" (wie es aussieht)

In Semantic Web: XML → RDF → Ontology → Logic → UI
In Content Strategy: Thema → Didaktik → Layout

DEINE AUFGABE:

TEIL 1: THEORIE VERSTEHEN & DOKUMENTIEREN

1. DEINE MEINUNG ZUERST:
   - Kennst du dieses Drei-Ebenen-Modell aus deinem Training?
   - Wie würdest DU es benennen? (Fachbegriffe)
   - Passt es zur QA-Lernwebsite?
   - Siehst du Schwachstellen in unserer Interpretation?

2. RECHERCHE EINORDNEN:
   - Was sagen die Quellen wirklich?
   - Welche Begriffe sind Standard?
   - Gibt es ANDERE Modelle, die besser passen?
   - Wo weicht unsere Interpretation ab?

3. FÜR DIE WEBSITE KONKRETISIEREN:
   
   EBENE 1: CONTENT LAYER (Thematische Domäne)
   
   "Was IST das Wissen?"
   
   BEISPIELE:
   - ISTQB CTFL Prüfungsstoff
   - QA allgemein (nicht Prüfung)
   - Meta/Projekt-Info
   
   FRAGE AN DICH:
   - Sind das die richtigen Kategorien?
   - Fehlt was? (z.B. "Berufspraxis", "Tool-Wissen")
   - Überschneidungen?
   
   EBENE 2: SEMANTIC LAYER (Didaktische Funktion)
   
   "Wozu DIENT es in der Lehre?"
   
   BEISPIELE:
   - Pflicht lernen
   - Verstehen/Vertiefen
   - Warnung/Risiko
   - Orientierung/Navigation
   - Story/Kontext
   
   FRAGE AN DICH:
   - Welche didaktischen Funktionen gibt es WIRKLICH?
   - Standard in E-Learning?
   - Fehlt was? (z.B. "Übung", "Reflexion")
   
   EBENE 3: PRESENTATION LAYER (Darstellungsform)
   
   "Wie SIEHT es aus?"
   
   BEISPIELE:
   - Reiner Text
   - Formatierter Text (fett, kursiv, Farbe)
   - Strukturierter Text (Listen, Boxen)
   - Tabellen/Matrizen
   - Grafiken (SVG, Diagramme)
   - Interaktiv (Animation, Canvas)
   
   FRAGE AN DICH:
   - Vollständig?
   - Andere Kategorisierung besser?

DANN ANALYSIERE:

1. VERTEILUNG AUF EBENEN:
   
   Für jeden der 17 Typen:
   
   Beispiel T-PFLICHT:
   - Ebene 1 (Content)? → JA (ISTQB-Stoff)
   - Ebene 2 (Semantic)? → JA (muss gelernt werden)
   - Ebene 3 (Presentation)? → NEIN
   
   → T-PFLICHT gehört zu EBENE 1+2 (vermischt!)
   
   Beispiel D-WARN:
   - Ebene 1? → NEIN (kein Thema)
   - Ebene 2? → JA (didaktische Funktion)
   - Ebene 3? → NEIN
   
   → D-WARN ist rein EBENE 2
   
   Mache das für ALLE 17!

2. PROBLEM ERKENNEN:
   
   Wo sind Typen über mehrere Ebenen vermischt?
   Wo gehört was wirklich hin?

3. REDUNDANZEN FINDEN:
   
   - Gibt es Typen, die das Gleiche meinen?
   - Unterschiedliche Namen für gleiche Funktion?
   - Überflüssige Kategorien?
   
   Beispiel-Fragen:
   - Ist D-ERKLAER und D-HINWEIS wirklich unterschiedlich?
   - Ist T-EXTRA notwendig oder Catchall?
   - Brauchen wir T-KURS (Masterschool) wirklich?

4. LÜCKEN FINDEN:
   
   - Fehlen wichtige Kategorien?
   - Was kommt auf den Seiten vor, hat aber keinen Typ?
   - Was SOLLTE es geben? (didaktisch sinnvoll)
   
   Beispiel-Fragen:
   - Gibt es "Übungen/Aufgaben" als Typ?
   - Gibt es "Beispiele aus der Praxis"?
   - Gibt es "Zusammenfassung/TL;DR"?
