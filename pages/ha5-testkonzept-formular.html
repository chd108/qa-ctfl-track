Projekt

Verantwortlich
QA-Team
1. Produktanalyse & Testumfang
Testobjekt
GroceryMate Online-Lebensmittel-Shop (grocerymate.masterschool.com)
Features im Test-Scope
GM-F01: Bewertungssystem (1-5 Sterne, opt. Text, Login-Pflicht)
GM-F02: Altersverifikation (Geburtsdatum-Modal, 18+, session-basiert)
GM-F03: Versandkosten (Freigrenze X€, Live-Update, Login nötig)
Hardware/Software-Anforderungen
Kategorie Spezifikation
Browser Chrome, Firefox, Safari, Edge (neueste 2 Versionen)
Geräte Desktop (Windows, macOS), Mobile (iOS, Android), Tablets
Backend grocerymate.masterschool.com, Session-Management (Cookie)
2. Teststrategie
Testarten
Funktionstest
Regressionstest
Grenzwerttest
Usability-Test
Kompatibilitätstest
Sicherheitstest
Testentwurfs-Techniken (CTFL 4.2)
Äquivalenzklassenbildung (EP)
Grenzwertanalyse (BVA)
Anwendungsfalltest
Fehlerermessen
Hauptrisiken & Maßnahmen
Risiko Maßnahme
Unklare Anforderungen (Freigrenze X€, Versandkosten Y€) Manuell ermitteln, mit PO abstimmen
Session-Persistenz unklar Tests: Browser-Refresh, Tab-Wechsel, Neustart
Duplikat-Bewertungen (Block vs. Update) Beide Szenarien testen, mit PO klären
3. Testziele
Primäre Ziele
  • Alle 3 Features funktionieren gemäß Spezifikation
  • Akzeptanzkriterien aus Testbasis-Analyse erfüllt
  • Keine Regression in bestehenden Features
  • Cross-Browser/Device Kompatibilität
Sekundäre Ziele
  • Modal-UI intuitiv verständlich
  • Live-Update <1 Sekunde
  • Keine XSS-Schwachstellen
  • Session-Daten sicher
Erwartete Ergebnisse (messbar)
Metrik Ziel
Testfall-Ausführungsquote ≥ 95%
Testfall-Erfolgsquote ≥ 90%
Kritische Fehler (Severity 1) 0
Hochprioritäre Fehler (Severity 2) Alle behoben
4. Testkriterien
Entry Criteria
  • Features deployed
  • Testbasis-Analyse freigegeben
  • Testfälle reviewed
  • Testumgebung bereit
  • Login funktioniert
  • Testdaten vorhanden
Exit Criteria
  • ≥95% Tests ausgeführt
  • ≥90% Tests bestanden
  • 0 Severity-1-Fehler
  • Max. 2 Severity-2-Fehler
  • Browser-Tests erfolgreich
  • UAT abgeschlossen
Suspension Criteria
  • Kritische Blocker
  • Umgebung instabil
  • Build fehlgeschlagen
  • >50% Team ausgefallen
  • >10 kritische Fehler
