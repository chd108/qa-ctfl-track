Testfallentwurf

HA5 · Testfallentwurf für drei neue Features im Kontext von CTFL Kapitel 4

Projekt: GroceryMate Features: 3 neue Funktionen Standard: CTFL 4.0.2 / IEEE 829 Testfälle: 28 (GM-F01: 9, GM-F02: 9, GM-F03: 10)
Einführung

Über dieses Dokument

Dieses Dokument enthält detaillierte Testfälle für die drei neuen Features von GroceryMate: Bewertungssystem (GM-F01), Altersverifikation (GM-F02) und Versandkosten (GM-F03). Jeder Testfall ist mit der verwendeten Testentwurfs-Technik gekennzeichnet und enthält Eingabedaten, Schritte und erwartete Ergebnisse.

Was sind Testfälle?
Testfälle sind konkrete Testanweisungen, die beschreiben, wie ein Feature getestet werden soll. Sie basieren auf der Testbasis-Analyse (HA4) und dem Testkonzept (HA5). Jeder Testfall hat:
  • Testentwurfs-Technik: Wie wurde der Test abgeleitet? (EP, BVA, etc.)
  • Eingabe: Welche Daten werden verwendet?
  • Schritte: Was wird getestet?
  • Erwartetes Ergebnis: Was sollte passieren?
Referenz: CTFL Kap. 4 (Testanalyse und -entwurf)
Warum diese Reihenfolge? EP → BVA → Use Case → Error Guessing
Die Testfälle sind nach CTFL 4.0.2-konformer Reihenfolge sortiert (Kap. 4.2): 1. Äquivalenzklassenbildung (EP) zuerst — EP definiert die Klassen (gültig/ungültig, eingeloggt/Gast). 2. Grenzwertanalyse (BVA) danach — BVA testet die Grenzen der Klassen. 3. Anwendungsfalltest (Use Case) — realistische Nutzerszenarien. 4. Fehlerermessen (Error Guessing) zuletzt — häufige Fehlerquellen. Für Lernzwecke und ISTQB-Prüfungen ist diese Reihenfolge didaktisch korrekt (Klassen → Grenzen → Szenarien → Fehler).

Verwendete Testentwurfs-Techniken

TechnikBeschreibungCTFL Kap.

Feature 1

Bewertungssystem (GM-F01)

GM-F01
Bewertungssystem für Produkte
TC-F01-001Äquivalenzklassenbildung (EP)
Bewertung als eingeloggter Nutzer (gültige Klasse)
Eingabe
Sterne: 4, Textfeld: "Sehr gute Qualität, schnelle Lieferung. Preis könnte besser sein.", Nutzer: eingeloggt
Schritte
  1. Als eingeloggter Nutzer auf Produktseite navigieren
  2. 4 Sterne auswählen
  3. Detailliertes Feedback im Textfeld eingeben
  4. "Bewertung abgeben" klicken
  5. Seite neu laden
  6. Prüfen, ob eigene Bewertung angezeigt wird
Erwartetes Ergebnis
Bewertung wird gespeichert. Nach Reload wird 4-Sterne-Bewertung mit Feedback-Text angezeigt. Durchschnittsbewertung berücksichtigt die neue Bewertung. Nutzername wird bei der Bewertung angezeigt.
TC-F01-002Äquivalenzklassenbildung (EP)
Bewertung als Gast (ungültige Klasse)
Eingabe
Sterne: 3, Textfeld: "Gutes Produkt", Nutzer: nicht eingeloggt (Gast)
Schritte
  1. Als Gast (nicht eingeloggt) auf Produktseite navigieren
  2. Versuchen, Bewertungsfunktion zu finden
Erwartetes Ergebnis
Bewertungsfunktion ist nicht sichtbar oder deaktiviert. Optional: Hinweis "Bitte einloggen, um zu bewerten" wird angezeigt.
TC-F01-003Grenzwertanalyse (BVA)
Bewertung mit genau 5 Sternen (Grenzwert)
Eingabe
Sterne: 5, Textfeld: leer, Nutzer: eingeloggt
Schritte
  1. Als eingeloggter Nutzer auf Produktseite navigieren
  2. 5 Sterne auswählen (maximaler Grenzwert)
  3. Textfeld leer lassen
  4. "Bewertung abgeben" klicken
Erwartetes Ergebnis
Bewertung wird erfolgreich gespeichert. 5-Sterne-Bewertung wird auf Produktseite angezeigt. Durchschnittsbewertung wird aktualisiert.
TC-F01-004Grenzwertanalyse (BVA)
Bewertung mit genau 1 Stern (Grenzwert)
Eingabe
Sterne: 1, Textfeld: "Produkt war schlecht", Nutzer: eingeloggt
Schritte
  1. Als eingeloggter Nutzer auf Produktseite navigieren
  2. 1 Stern auswählen (minimaler Grenzwert)
  3. Feedback-Text eingeben
  4. "Bewertung abgeben" klicken
Erwartetes Ergebnis
Bewertung wird erfolgreich gespeichert. 1-Stern-Bewertung + Text werden angezeigt. Durchschnittsbewertung sinkt.
TC-F01-005Grenzwertanalyse (BVA)
Textfeld mit maximaler Zeichenlänge (falls definiert)
Eingabe
Sterne: 3, Textfeld: Text mit X Zeichen (Grenzwert), Nutzer: eingeloggt
Schritte
  1. Als eingeloggter Nutzer auf Produktseite navigieren
  2. 3 Sterne auswählen
  3. Text mit genau X Zeichen eingeben (X = max. Zeichenlänge)
  4. "Bewertung abgeben" klicken
Erwartetes Ergebnis
Bewertung wird akzeptiert, wenn X innerhalb des Limits liegt. Falls Limit überschritten: Fehlermeldung oder Eingabe wird gekürzt. → Maximale Zeichenanzahl laut HA4 noch nicht spezifiziert!
TC-F01-006Anwendungsfalltest (Use Case)
Durchschnittsbewertung wird korrekt berechnet
Eingabe
Produkt hat bereits Bewertungen: 5★, 4★, 3★. Neue Bewertung: 2★
Schritte
  1. Produkt mit bestehenden Bewertungen aufrufen
  2. Aktuelle Durchschnittsbewertung notieren (sollte 4.0 sein)
  3. Als eingeloggter Nutzer 2 Sterne vergeben
  4. "Bewertung abgeben" klicken
  5. Neue Durchschnittsbewertung prüfen
Erwartetes Ergebnis
Neue Durchschnittsbewertung = (5+4+3+2) / 4 = 3.5 Sterne. → Rundungsregel laut HA4 noch zu klären!
TC-F01-007Fehlerermessen (Error Guessing)
Bewertung ohne Sternauswahl (nur Text)
Eingabe
Sterne: keine Auswahl (0), Textfeld: "Tolles Produkt!", Nutzer: eingeloggt
Schritte
  1. Als eingeloggter Nutzer auf Produktseite navigieren
  2. KEINE Sterne auswählen
  3. Nur Feedback-Text eingeben
  4. "Bewertung abgeben" klicken
Erwartetes Ergebnis
System zeigt Fehlermeldung "Bitte wählen Sie mindestens 1 Stern" oder ähnlich. Bewertung wird NICHT gespeichert.
TC-F01-008Fehlerermessen (Error Guessing)
Zweite Bewertung desselben Produkts (Duplikat)
Eingabe
Sterne: 4, Textfeld: "Zweiter Versuch", Nutzer: eingeloggt (hat bereits bewertet)
Schritte
  1. Als eingeloggter Nutzer Produkt bereits bewertet haben
  2. Erneut auf Produktseite navigieren
  3. Versuchen, zweite Bewertung abzugeben (4 Sterne + Text)
Erwartetes Ergebnis
Entweder: (a) System blockiert mit Fehlermeldung "Sie haben dieses Produkt bereits bewertet", ODER (b) Bestehende Bewertung wird aktualisiert. → Zu verifizieren laut offenen Punkten HA4!
TC-F01-009Fehlerermessen (Error Guessing)
XSS-Schutz im Textfeld (Sicherheit)
Eingabe
Sterne: 5, Textfeld: "<script>alert('XSS')</script>Tolles Produkt", Nutzer: eingeloggt
Schritte
  1. Als eingeloggter Nutzer auf Produktseite navigieren
  2. 5 Sterne auswählen
  3. JavaScript-Code im Textfeld eingeben (XSS-Versuch)
  4. "Bewertung abgeben" klicken
  5. Prüfen, ob Code ausgeführt wird
Erwartetes Ergebnis
System bereinigt die Eingabe (Sanitization). Script wird NICHT ausgeführt. Text wird escaped dargestellt oder Eingabe wird blockiert.
Automatisierungempfehlung: GM-F01
Zu automatisieren: TC-F01-003, 004 (Grenzwerte 5/1 Stern), TC-F01-002 (Gast-Zugriff), TC-F01-009 (XSS-Schutz), TC-F01-006 (Durchschnittsberechnung). Manuell: TC-F01-008 (Duplikat — Verhalten unklar), TC-F01-001 (UX), TC-F01-005 (Textlänge noch unbekannt).
Feature 2

Altersverifikation (GM-F02)

GM-F02
Altersverifikation für alkoholische Produkte
TC-F02-001Äquivalenzklassenbildung (EP)
Geburtsdatum über 18 Jahre (gültige Klasse)
Eingabe
Geburtsdatum: 01.01.2000 (26 Jahre alt)
Schritte
  1. Auf /store navigieren
  2. Modal erscheint
  3. Geburtsdatum eingeben: deutlich über 18 Jahren
  4. "Bestätigen" klicken
Erwartetes Ergebnis
Modal schließt sich. Store-Seite wird vollständig angezeigt. Zugriff gewährt.
TC-F02-002Äquivalenzklassenbildung (EP)
Geburtsdatum unter 18 Jahren (ungültige Klasse)
Eingabe
Geburtsdatum: Heute minus 17 Jahre und 364 Tage
Schritte
  1. Auf /store navigieren
  2. Modal erscheint
  3. Geburtsdatum eingeben: 1 Tag unter 18 Jahren
  4. "Bestätigen" klicken
Erwartetes Ergebnis
Zugriff verweigert. Hinweistext "Sie müssen mindestens 18 Jahre alt sein" oder ähnlich. Store-Produkte werden NICHT angezeigt.
TC-F02-003Grenzwertanalyse (BVA)
Geburtsdatum genau 18 Jahre (Grenzwert)
Eingabe
Geburtsdatum: Heute minus genau 18 Jahre
Schritte
  1. Auf /store navigieren
  2. Modal mit "Please enter your birth date" erscheint
  3. Geburtsdatum eingeben: genau 18 Jahre alt
  4. "Bestätigen" klicken
Erwartetes Ergebnis
Modal schließt sich. Store-Seite mit alkoholischen Produkten wird angezeigt. Zugriff gewährt.
TC-F02-004Grenzwertanalyse (BVA)
Zukünftiges Geburtsdatum (ungültiger Grenzwert)
Eingabe
Geburtsdatum: 01.01.2030 (in der Zukunft)
Schritte
  1. Auf /store navigieren
  2. Modal erscheint
  3. Zukünftiges Datum eingeben
  4. "Bestätigen" klicken
Erwartetes Ergebnis
System zeigt Fehlermeldung "Geburtsdatum kann nicht in der Zukunft liegen" oder verhindert Eingabe. Zugriff wird nicht gewährt.
TC-F02-005Anwendungsfalltest (Use Case)
Modal erscheint nur beim ersten Aufruf (Session-basiert)
Eingabe
Geburtsdatum: 01.01.1990 (über 18), Session: aktiv
Schritte
  1. Auf /store navigieren (erstmals in Session)
  2. Modal erscheint, Geburtsdatum eingeben (über 18)
  3. Store-Seite wird angezeigt
  4. Auf andere Seite navigieren (z.B. Home)
  5. Zurück zu /store navigieren (innerhalb derselben Session)
Erwartetes Ergebnis
Modal erscheint beim zweiten Aufruf NICHT erneut. Store-Seite wird direkt angezeigt.
TC-F02-006Anwendungsfalltest (Use Case)
Modal erscheint nach Browser-Neustart erneut
Schritte
  1. Auf /store navigieren, Modal mit Geburtsdatum ausfüllen (über 18)
  2. Browser komplett schließen und neu starten
  3. Erneut auf /store navigieren
Erwartetes Ergebnis
Modal erscheint ERNEUT, da Session beendet wurde. → Session-Dauer laut HA4 noch zu klären!
TC-F02-007Anwendungsfalltest (Use Case)
Modal in neuem Tab/Fenster
Schritte
  1. In Tab 1: Auf /store navigieren, Modal ausfüllen (über 18)
  2. Neuen Tab öffnen
  3. In Tab 2: Auf /store navigieren
Erwartetes Ergebnis
Modal erscheint in Tab 2 NICHT erneut, wenn Session geteilt wird. Andernfalls: Modal erscheint erneut. → Verhalten laut HA4 noch zu verifizieren!
TC-F02-008Fehlerermessen (Error Guessing)
Ungültiges Datum (z.B. 31. Februar)
Eingabe
Geburtsdatum: 31.02.2000 (ungültig)
Schritte
  1. Auf /store navigieren
  2. Modal erscheint
  3. Ungültiges Datum eingeben
  4. Versuchen, zu bestätigen
Erwartetes Ergebnis
System zeigt Fehlermeldung "Ungültiges Datum" oder verhindert Eingabe. → Verhalten laut HA4 noch zu verifizieren!
TC-F02-009Fehlerermessen (Error Guessing)
Leeres Datumsfeld
Eingabe
Geburtsdatum: leer (keine Eingabe)
Schritte
  1. Auf /store navigieren
  2. Modal erscheint
  3. KEIN Datum eingeben
  4. "Bestätigen" klicken
Erwartetes Ergebnis
System zeigt Fehlermeldung "Bitte geben Sie Ihr Geburtsdatum ein" oder ähnlich. Zugriff wird nicht gewährt.
Automatisierungempfehlung: GM-F02
Zu automatisieren: TC-F02-003, 002 (Grenzwert 18, unter 18), TC-F02-001 (gültige Klasse), TC-F02-009 (leeres Feld), TC-F02-005 (Session-Persistenz). Manuell: TC-F02-004 (ungültiges Datum), TC-F02-007, 006 (Browser/Tabs), TC-F02-009 (zukünftiges Datum).
Feature 3

Versandkosten & Freigrenze (GM-F03)

GM-F03
Versandkosten & Freigrenze
TC-F03-001Äquivalenzklassenbildung (EP)
Checkout als Gast (ohne Login)
Eingabe
Warenkorb: Produkte im Wert > X€, Nutzer: nicht eingeloggt (Gast)
Schritte
  1. Als Gast Produkte in Warenkorb legen
  2. Versuchen, zu /checkout zu navigieren
Erwartetes Ergebnis
Weiterleitung zu /auth (Login-Seite). Checkout ist ohne Login nicht möglich. → Laut HA4: Checkout nur für eingeloggte Nutzer!
TC-F03-002Äquivalenzklassenbildung (EP)
Sehr niedriger Bestellwert (deutlich unter Freigrenze)
Eingabe
Warenkorb: 1 Produkt à 5€, Nutzer: eingeloggt
Schritte
  1. Als eingeloggter Nutzer günstiges Produkt (5€) in Warenkorb
  2. Zu /checkout navigieren
  3. Versandkosten prüfen
Erwartetes Ergebnis
Versandkosten = Y€. Hinweis "Noch X - 5€ bis zur kostenlosen Lieferung" wird angezeigt.
TC-F03-003Äquivalenzklassenbildung (EP)
Sehr hoher Bestellwert (deutlich über Freigrenze)
Eingabe
Warenkorb: Produkte im Wert von 200€, Nutzer: eingeloggt
Schritte
  1. Als eingeloggter Nutzer viele Produkte (200€ Gesamtwert) in Warenkorb
  2. Zu /checkout navigieren
  3. Versandkosten prüfen
Erwartetes Ergebnis
Versandkosten = 0€. Freigrenze wurde deutlich überschritten.
TC-F03-004Grenzwertanalyse (BVA)
Bestellwert genau an Freigrenze X€ (Grenzwert)
Eingabe
Warenkorb: Produkte im Wert von genau X€ (Freigrenze), Nutzer: eingeloggt
Schritte
  1. Als eingeloggter Nutzer Produkte im Wert von genau X€ in Warenkorb legen
  2. Zu /checkout navigieren
  3. Versandkosten prüfen
Erwartetes Ergebnis
Versandkosten = 0€. → Freigrenze X laut HA4 noch zu ermitteln!
TC-F03-005Grenzwertanalyse (BVA)
Bestellwert X − 0,01€ (Grenzwert unter Freigrenze)
Schritte
  1. Als eingeloggter Nutzer Produkte im Wert von X − 0,01€ in Warenkorb legen
  2. Zu /checkout navigieren
  3. Versandkosten prüfen
Erwartetes Ergebnis
Versandkosten = Y€. Hinweis "Noch 0,01€ bis zur kostenlosen Lieferung" oder ähnlich. → Versandkosten Y laut HA4 noch zu ermitteln!
TC-F03-006Grenzwertanalyse (BVA)
Bestellwert X + 0,01€ (Grenzwert über Freigrenze)
Schritte
  1. Als eingeloggter Nutzer Produkte im Wert von X + 0,01€ in Warenkorb legen
  2. Zu /checkout navigieren
  3. Versandkosten prüfen
Erwartetes Ergebnis
Versandkosten = 0€. Freigrenze wurde erreicht.
TC-F03-007Anwendungsfalltest (Use Case)
Live-Update bei Warenkorbänderungen
Schritte
  1. Als eingeloggter Nutzer Produkte im Wert von 20€ in Warenkorb (< X€)
  2. Zu /checkout navigieren, Versandkosten prüfen (sollte Y€ sein)
  3. Zurück zum Warenkorb, weiteres Produkt hinzufügen (Gesamtwert > X€)
  4. Zu /checkout zurückkehren, Versandkosten erneut prüfen
Erwartetes Ergebnis
Versandkosten ändern sich SOFORT von Y€ auf 0€. Keine Seiten-Refresh nötig.
TC-F03-008Anwendungsfalltest (Use Case)
Live-Update bei Mengenänderung
Eingabe
Warenkorb: 1× Produkt à 30€, dann Menge auf 2 erhöhen
Schritte
  1. Als eingeloggter Nutzer 1× Produkt (30€) in Warenkorb
  2. Zu /checkout navigieren (Versandkosten = Y€)
  3. Menge des Produkts auf 2 erhöhen (Gesamtwert: 60€)
  4. Versandkosten prüfen
Erwartetes Ergebnis
Versandkosten aktualisieren sich automatisch auf 0€, sobald Freigrenze überschritten.
TC-F03-009Anwendungsfalltest (Use Case)
Produkt entfernen → Versandkosten wieder aktiv
Schritte
  1. Als eingeloggter Nutzer Produkte im Wert > X€ in Warenkorb
  2. Zu /checkout navigieren (Versandkosten = 0€)
  3. Ein Produkt entfernen, sodass Gesamtwert < X€ fällt
  4. Versandkosten prüfen
Erwartetes Ergebnis
Versandkosten ändern sich SOFORT von 0€ auf Y€. Live-Update funktioniert bidirektional.
TC-F03-010Fehlerermessen (Error Guessing)
Leerer Warenkorb im Checkout
Eingabe
Warenkorb: leer (0 Produkte), Nutzer: eingeloggt
Schritte
  1. Als eingeloggter Nutzer direkt zu /checkout navigieren (ohne Produkte im Warenkorb)
Erwartetes Ergebnis
System zeigt sinnvollen Hinweis "Ihr Warenkorb ist leer" und verhindert Checkout. Kein Absturz. → Verhalten laut HA4 noch zu verifizieren!
Automatisierungempfehlung: GM-F03
Grenzwert-Tests (TC-F03-004, 005, 006) und EP-Tests (001, 002, 003) gut automatisierbar. Live-Update (007, 008, 009) je nach Tool. TC-F03-010 (leerer Warenkorb) manuell prüfen, bis Verhalten geklärt.

Warum 68% Automatisierung?
Grenzwert-Tests sind kritische Geschäftslogik und müssen häufig geprüft werden. Regression ist wichtig (Login, Checkout). Mathematische Logik (Bewertungs-Durchschnitt, Preisberechnung) ist ideal für Automatisierung. XSS-Schutz sollte in CI/CD laufen. Manuelle Tests bleiben wichtig für: UX, unklare Anforderungen (offene Punkte HA4), explorative Tests.
